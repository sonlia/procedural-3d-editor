# Web 3D 编辑器设计方案（完整版）

> 整体方向：**去重、保持核心、统一到 JS 代码拼接模型**
>
> 本文档整合了从初始需求 → V2 收敛 → 视口打磨三个阶段的所有讨论结果。

---

## 〇、项目概述（初始需求）

### 项目目标
开发一款基于 Web 的 3D 编辑器，核心特色是采用**节点图驱动**的工作流，灵感来源于 Houdini 等专业 DCC 软件。用户通过拖拽和连接节点来定义场景、模型、材质、光照、物理模拟、后期处理等，最终生成可视化的 3D 场景。

### 核心特色
- **节点图 = 一切**：graph 包含所有的 Three.js 配置与逻辑，通过节点连接实现。
- **代码拼接执行**：节点 = JS 代码生成器，拼接后 `new Function` 编译执行。
- **Houdini 风格**：SOP/Material/COP/DOP 分类，Display Flag，Subgraph。

### 技术栈
- **框架**：Next.js 16 + TypeScript + Tailwind CSS 4 + shadcn/ui
- **3D 渲染**：Three.js 0.185
- **节点图**：用户重写版 LiteGraph（`src/lib/editor/litegraph.js`）
- **代码拼接策略**：`src/lib/editor/codeStrategies/index.js`
- **状态管理**：Zustand
- **面板布局**：react-resizable-panels

## 一、节点图系统（精简版）

### 保留
- **Display Flag**：每条链路上标记视口显示哪个节点结果（Houdini 蓝点逻辑），同一链路同时只有一个 display。
- **强类型端口**：`geometry / material / texture / number / vector / color / matrix / image / camera / light / body(物理) / time`，类型不匹配需 Convert 节点。
- **节点删除/连接/参数编辑**：用 LiteGraph 默认交互即可。

### 去掉
- ~~Bypass/Mute~~：因为求值是 JS 拼接 + 编译执行，可以直接改图重编，不需要旁路机制。
- ~~节点缓存与惰性求值~~：由 LiteGraph 自己处理。
- ~~For-Each / Switch / Merge 控制流节点~~：MVP 不做。
- ~~Subnet / HDA 封装~~：不做，全平铺。

### 计算层定位
- 所有节点的求值逻辑最终拼接成一段 JS 源码字符串。
- 拓扑排序 → 每个节点贡献一段 JS 片段 → 拼成完整函数 → `new Function` 执行。
- 这意味着"节点 = JS 代码生成器"，包括属性 wrangle 也用 JS 表达式（不是 VEX）。

### 两种代码产物
- **jsCode**：声明式拼接代码（定义 `__out_<safeVar>` 变量，供下游节点引用）
- **bgJsCode**：运行时执行代码（直接操作 THREE 对象，存到 `ctx.__meshes` / `ctx.__geometries` 等缓存）

## 二、属性系统（核心保留）

### 四类属性
| 类 | 作用域 | 例子 |
|---|---|---|
| Detail | 整个几何体一份 | 总面积、版本、元数据 |
| Prim（面） | 每个面一份 | 材质 ID、面法线、组归属 |
| Point（点） | 每个点一份 | 位置、速度、颜色、自定义 |
| Vertex（顶点） | 每个面的每个角一份 | UV、顶点法线、顶点色 |

### 属性编辑节点（用 JS 表达式）
- 提供 Attribute Wrangle 节点，参数是一个 JS 代码框。
- 上下文注入：`P / N / Cd / v / uv / time / frame / id` 等。
- 例子：`Cd.y = P.y * 2; P.x += sin(time) * 0.1;`
- 拼接后生成的 JS 直接对 geometry 的 typed array 操作。

### Group 节点选择条件
支持：
- 点/线/面 ID
- 位置范围（bbox）
- 法线方向（如 `N.y > 0.5`）
- 面积阈值
- JS 表达式（`P.y > 0 && N.y > 0.5`）
- 邻近性（距离某物体/曲线 N 米内）
- 已有分组（合并/差集/交集）

### 属性生命周期
- Attribute Delete / Rename / Copy 三个节点管理属性传递与清理。

## 三、Timeline / 动画系统

### 时间来源
- Three.js 自带 `clock`，注入到求值上下文为 `time` / `frame` 全局变量。
- 不需要单独的"时间节点"。

### Key 帧机制
- 任何数值属性右键可 "Key Frame"。
- Key 后该属性在 timeline 上出现一个"调节点"（关键帧）。
- 关键帧默认贝塞尔插值，可切换 Linear / Step / Constant。

### Timeline 显示规则
| 选中状态 | Timeline 显示 |
|---|---|
| 未选中任何节点 | 默认显示**所有节点**的所有 key 过的属性曲线 |
| 选中单个节点 | 只显示该节点的属性和曲线 |
| 多选节点 | 显示所选节点的属性和曲线 |

### Timeline 上的操作
- 拖动关键帧、复制粘贴关键帧、删除关键帧。
- 缩放时间区域（框选放大）。
- 多曲线叠加显示（每个属性一条颜色）。
- 播放/暂停/循环/帧步进/跳到首尾帧。

### 动画驱动
- 不单独做 Driver/Driven 子系统。
- 通过 JS 代码节点直接引用：`light.intensity = mesh.scale.x * 2;`
- 自然落在代码拼接模型里。

- ~~动画层~~
- ~~动画剪辑与状态机~~
- ~~烘焙到点缓存~~

## 四、视口交互（最终版 — 核心打磨区）

### 导航
- **左键**：选中（点/线/面/物体，按当前模式）
- **Alt + 左键**：旋转
- **Alt + 右键**：平移
- **中键**：平移（保留习惯）
- **右键（无 Alt）**：无操作
- **滚轮**：缩放
- **F**：聚焦选中
- **A**：聚焦全部

### 不做
- ~~正交/透视切换~~
- ~~视口书签~~

### 变换 Gizmo
- W/E/R 切换 移动/旋转/缩放。
- 选中物体后显示三轴 Gizmo，可拖拽。
- 摄像机节点选中后也能用 Gizmo 调位置。

### 摄像机
- 默认有一个**自由摄像机**用于视口导航（不属于节点图）。
- 节点图中可添加 Camera 节点，作为渲染输出相机。
- Camera 节点选中后，Gizmo 可调整其位置/朝向。
- 同一时刻只有一个渲染相机生效。
- **Camera 选择器用 Three.js 实现**（不用 HTML 下拉框），叠加在视口右上角。

### 显示模式
实体 / 线框 / 点 / X 射线 / 法线 / 隔离查看（Isolate）。
- **显示模式选择器用 Three.js 实现**（不用 HTML 下拉框），叠加在视口右上角，和性能统计（FPS/Tris/Calls）一样作为 overlay 渲染。

### 选择模式
- **Object / Component 切换用 Three.js 实现**（不用 HTML 按钮），叠加在视口右上角。
- 保留：单选 + Shift 加选 + Ctrl 减选 + 框选（当前模式下生效）。
- 选择高亮用线框叠加色，不改材质。
- 不做选择集、选择历史、UV 选择模式切换。

### 点/线/面选择（Component 模式）
- **点（Vertex）选择**：raycast 命中 mesh 后，找最近的顶点，高亮显示为一个发光小球。
- **线（Edge）选择**：raycast 命中后，找最近的边（两个顶点之间），高亮显示为加粗线段。
- **面（Face）选择**：raycast 命中后，高亮显示该三角面。
- 选择的点/线/面 ID 写入 store，供下游 Group/Wrangle 节点引用。
- 框选时根据当前子模式（点/线/面）框选对应元素。

### 默认灯光（关键设计）
- **无灯光时自动添加默认灯光**：当场景中没有任何 Light 节点时，视口自动添加一个默认的方向光 + 环境光，确保模型可见。
- **添加灯光后自动移除默认灯光**：一旦用户在 graph 中添加了任意 Light 节点（无论是否连接），默认灯光自动删除。
- **判断逻辑**：检查 graph 中是否存在任何 `type` 包含 `light` 的节点（不需要看是否连接到 SceneOutput）。
- **实现**：默认灯光是视口 host scene 的成员，不属于节点图，不参与序列化。

## 五、材质系统

### 底层
- 用 Three.js 原生 Material 节点系统（MeshStandard / Physical / Basic / Shader 等）。
- 通过 Material 节点的输入端口接收贴图、颜色、数值。
- 暂不引入 TSL，保持代码拼接模型统一。

### 材质多层（Material Layers）
- 支持，类似 COP 操作：Base + Coat + Emissive 等多层叠加。
- 每层是一个材质节点，通过 Layer 节点组合。

### 材质应用
- Material 节点输出 → Model 节点 / Geometry 节点的 `material` 端口。
- 一个 Model 支持多个 material slot（对应 GLTF 的 multi-mesh）。
- Material 节点可指定 slot index。

### 不做
- ~~材质预览球独立视口~~：直接在主视口实时看效果即可。

## 六、COPs 合成节点

### 节点类型
| 类别 | 节点 |
|---|---|
| 渲染通道 | Beauty / Depth / Normal / AO / Albedo |
| 色彩 | LUT / Color Correction / Levels / Curves |
| 模糊锐化 | Gaussian Blur / Bloom / DOF / Motion Blur |
| 合成 | Add / Screen / Overlay / Mask / Channel Merge |
| 输出 | Output to Screen |

### 求值方式
- 与 SOP/Material 一致，通过 JS 拼接，调用 Three.js 的 `EffectComposer` + `Pass` API。
- 实时后处理走 GPU pass，离线合成走 CPU 图像处理。

## 八、撤销/重做（精简）

- 单一撤销栈，所有操作（节点增删、参数改、连接改、属性改）入栈。
- 视口导航操作不入栈（避免 Ctrl+Z 回退视角）。
- 不做历史栈可视化面板、不做原子操作合并。

## 十一、待确认的最后几点

1. **代码拼接的执行模型**：每帧重新拼+编译，还是只在图变化时拼+编译、运行时只传参？倾向**后者**——参数变化走 uniform，结构变化才重编。
2. **JS 表达式节点的安全边界**：用 `new Function` 在主线程跑，还是开 Web Worker？前者快但可能卡 UI，后者隔离但通信开销。
3. **Display Flag 与求值关系**：display flag 切换是否触发重编？还是只是切换视口显示对象？
4. **材质多层**的具体叠加方式：是不是 COP 风格（每层有 blend mode + mask）？还是简单的优先级覆盖？

### Scotty3D 核心架构

Scotty3D 是 C++ 项目，不能直接在浏览器运行。核心价值在于：

   - `split_edge`: 边中点分裂
   - `flip_edge`: 边翻转
   - `collapse_edge`: 边坍缩
   - `extrude_face`: 面挤出

   - `triangulate`: 三角化
   - `linear_subdivide`: 线性细分
   - `catmark_subdivide`: Catmull-Clark 细分
   - `remesh`: 重网格化
   - `simplify`: 网格简化

4. **ID-based Picking** (`src/gui/model.cpp`)
   - 每个顶点/边/面分配唯一 ID
   - 渲染到离屏 buffer（颜色编码 ID）
   - 读像素反查元素
   - 比射线相交+距离比较更精确
